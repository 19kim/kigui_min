프로젝트 추가

해당 프로젝트를 이용하기 위해서는 kigui.css, kigui.js, kigui-custom.css(선택) 세 가지를 포함하여야 한다.

프로젝트는 웹으로도 추가할 수 있다.
<link href="http://minkigui.kimig.net/kigui.css" rel="stylesheet" />
<script src="http://minkigui.kimig.net/kigui.js" type="text/javascript"></script> (github를 통해서 가져오므로, github에 문제가 생길 시 사이트에 이상이 생길 수 있다.)

자바스크립트 객체

kigui는 자바 스크립트 객체를 통해서 기능 조절이 일부 포함되어있다.
kigui 객체는 다음과 같은 구조와 기능을 한다.
kigui
  • (obj)layout
    • (obj) body
      • (val)element : 해당 객체를 저장해둔다.
      • (func)getElement : 해당 객체를 불러온다.
      • (func)enableScroll : 스크롤을 활성화 한다.
      • (func)disableScroll : 스크롤을 고정시킨다.
    • (obj)header
      • (obj)msg
        • (obj)text
          • (val)element : 해당 객체를 저장해둔다.
          • (func)getElement : 해당 객체를 불러온다.
          • (func)setText(text) : 메시지의 텍스트를 설정한다.
        • (obj)close
          • (val)element : 해당 객체를 저장해둔다.
          • (func)getElement : 해당 객체를 불러온다.
        • (val)code : 메시지의 코드를 저장한다.
        • (val)element : 해당 객체를 저장해둔다.
        • (func)getElement : 해당 객체를 불러온다.
        • (func)show : 메시지를 보여준다.
        • (func)hide : 메시지를 숨긴다.
        • (func)toggle : 메시지의 상태를 토글 한다.(show, hide)
        • (func)setCode(string) : 코드를 설정한다.
        • (func)getCode : 코드를 반환 한다.
        • (func)loadCode : 로컬 저장소에 저장된 코드를 불러온다.
        • (func)isNewMsg : 현재 지정된 코드와 로컬 저장소의 코드를 비교한다.
        • (func)init(text, code) : 설정한다.
        • (func)setEvent(text, code, checkNewMsg) : 시작 시 생성될 메시지 이벤트를 추가한다.
      • (val)element : 해당 객체를 저장해둔다.
      • (func)getElement : 해당 객체를 불러온다.
    • (obj)sidemenu
      • (val)element : 해당 객체를 저장해둔다.
      • (func)getElement : 해당 객체를 불러온다.
      • (func)show : 사이드 메뉴를 연다.
      • (func)hide : 사이드 메뉴를 닫는다.
      • (func)eventMenuOpen : 사이드 메뉴를 여는 이벤트(리스너에 바로 추가)
      • (func)eventMenuClose : 사이드 메뉴를 닫는 이벤트(리스너에 바로 추가)
    • (obj)others
      • (obj)msg
        • (val)element : 해당 객체를 저장해둔다.
        • (func)getElement : 해당 객체를 불러온다.
      • (obj)pageCover
        • (val)element : 해당 객체를 저장해둔다.
        • (func)getElement : 해당 객체를 불러온다.
        • (func)show : 페이지 커버를 활성화 한다.
        • (func)hide : 페이지 커버를 숨긴다.
      • (val)element : 해당 객체를 저장해둔다.
      • (func)getElement : 해당 객체를 불러온다.
  • (obj)object (현재 기능이 존재하지 않음.)
getElement가 존재하는 객체는 굳이 객체 찾기를 하지 않아도 된다.
객체를 찾은 적이 없으면 찾아서 element변수에 담아두고, 다음부터는 변수값을 반환하는 방식이다.

헤더

헤더는 웹 페이지에서 상단 부분을 나타내는 영역이다.
<header class="kigui-l-h">

</header>
헤더의 영역을 지정해주는 태그이며, 이 내부에 액션바와 메시지가 들어간다.
헤더에는 추가적인 속성이 있으며, 해당 클래스를 해당 태그에 부여하면 속성이 부여된다.

kigui-l-h-s : 헤더에 그림자를 추가한다.
kigui-l-h-f : 헤더를 공중에 띄운다. 단, 아래의 태그가 헤더 태그 다음에 위치하여야 한다.
<div class="kigui-l-o-h"></div>

액션 바

(이 태그를 이용하려면, 머티리얼 폰트를 사용하여야 합니다.)

액션바는 헤더 내에서 상단에 위치한다.
타이틀과 메뉴 버튼 등이 위치할 수 있다.
<div class="kigui-l-h-a">
    <h1 class="kigui-l-h-a-t">타이틀</h1>
    <button class="kigui-l-h-a-b" id="btnMenu">
        <i class="material-icons">menu</i>
    </button>
</div>
타이틀 부분에 원하는 제목을 작성한다.

메시지

(이 태그를 이용하려면, 머티리얼 폰트를 사용하여야 합니다.)

메시지는 헤더 내에서 액션바 다음에 위치한다.
메시지 기능은 헤더의 액션바 밑으로 출력이되는 알림이다.
<div class="kigui-l-h-m">
    <div class="kigui-l-h-m-t"></div>
    <div class="kigui-l-h-m-c">
        <i class="material-icons">close</i>
    </div>
</div>
메시지를 사용하기 위해서는 헤더 태그 다음에 다음 태그가 위치하여야 한다.
<div class="kigui-l-o-m"></div>
또한, 이 태그는 자바 스크립트가 필수적으로 연계된다.
kigui.layout.header.msg 객체 부분을 참고한다.

전체 태그

헤더의 전체 기능을 사용한 소스이다.
<header class="kigui-l-h kigui-l-h_s kigui-l-h_f">
    <div class="kigui-l-h-a">
        <h1 class="kigui-l-h-a-t">타이틀</h1>
        <button class="kigui-l-h-a-b" id="btnMenu">
            <i class="material-icons">menu</i>
        </button>
    </div>
    <div class="kigui-l-h-m">
        <div class="kigui-l-h-m-t"></div>
        <div class="kigui-l-h-m-c">
            <i class="material-icons">close</i>
        </div>
    </div>
</header>

사이드 메뉴

사이드 메뉴는 왼쪽에서 숨겨져있다가 버튼을 누르면 나오는 메뉴이다.
필수적으로 자바 스크립트를 사용하여야 한다.
<aside id="sideMenu" class="kigui-l-s kigui-l-s_c">
    <div class="kigui-l-s-h">
        <div>타이틀</div>
        <div>내용</div>
    </div>
    <div class="kigui-l-s-b">
        <div class="kigui-l-s-b-c">
            <ul>
                <li>
                    <h3>카테고리 1</h3>
                    <ul>
                        <li><!--항목--></li>
                        <li><!--항목--></li>
                        <li><!--항목--></li>
                        <li><!--항목--></li>
                        <li><!--항목--></li>
                    </ul>
                </li>
                <li>
                    <h3>카테고리 2</h3>
                    <ul>
                        <li><!--항목--></li>
                        <li><!--항목--></li>
                        <li><!--항목--></li>
                        <li><!--항목--></li>
                        <li><!--항목--></li>
                    </ul>
                </li>
            </ul>
        </div>
    </div>
</aside>
사이드 메뉴는 위 소스코드를 본인이 맞추어서 변경하여서 사용하여야 한다.
kigui-l-s-h(사이드메뉴 헤더) 헤더 부분은 본인의 필요에 따라서 삭제하여도 된다.
kigui-l-s-b-c(사이드 메뉴 카테고리) 카테고리 부분은 본인이 추가할 항목에 따라서 조절한다.
그리고 카테고리 최 하단 li태그 내에 주석 부분은 다음 항목들을 사용해야 한다.

그리고 아래의 태그를 body 내부에 반드시 추가한다.
<div class="kigui-l-o-p"></div>

일반 링크

일반적인 링크 항목이며, 간단한 형태를 띄고 있다. a태그의 href속성에 원하는 링크를 넣거나, id속성을 부여하고 자바스크립트 이벤트를 추가한다.
<a href="#">항목</a>

스위치

스위치 항목이며, 항목을 클릭 할 시 스위치가 토글 된다.
<label for="아이디">
    <div>스위치 항목</div>
    <label class="kigui-o-s">
        <input type="checkbox" id="아이디" />
        <div class="kigui-o-s-l"></div>
        <div class="kigui-o-s-b"></div>
    </label>
</label>
ON/OFF 기본 상태는 input태그에 checked 속성을 추가하여야 합니다.
'아이디' 부분은 직접 적어주셔야 합니다.

컨텐츠

컨텐츠가 포함되는 영역이다. 현재까지는 별다른 스타일이나 스크립트가 적용되지 않았으나, 일단 지정을 해두는 것을 권장한다.
<section class="kigui-l-c">

</section>

프레임

프레임은 내용물을 구분하여 담기 위한 공간입니다.

큰 제목

내용

작은 제목

내용

작은 제목

내용
<div class="kigui-o-f">
    <h3>큰 제목</h3>
    내용
    <h4>작은 제목</h4>
    내용
    <hr />
    <h4>작은 제목</h4>
    내용
</div>
h3 : 큰 제목
h4 : 작은 제목
hr : 가로 구분 줄

입력 공간

입력 공간은 프레임에서 바디 내에 입력을 받을 때. 어울리도록 짜놓은 것 입니다.
[타이틀]
label태그로 덮혀 있어 넓은 해당 항목의 넓은 범위 아무곳이나 눌러도 텍스트 박스가 선택 됩니다.

아래는 일반 한 줄 텍스트를 입력 받을 수 있다.
<label for="아이디" class="kigui-o-f-b-i">
    <div>
        텍스트 박스
    </div>
    <div>
        <input id="아이디" class="kigui-o-t"/>
    </div>
</label>
아래는 여러 줄의 텍스트를 입력받을 수 있다.
<label for="아이디" class="kigui-o-f-b-i">
    <div>
        텍스트 박스 2 (멀티 라인)
    </div>
    <div>
        <textarea id="아이디" class="kigui-o-t"></textarea>
    </div>
</label>

버튼

기본적인 버튼입니다.
button 태그와 a태그 둘다 사용할 수 있도록 제작되었습니다.
<button class="kigui-o-b-n">버튼</button>

텍스트

input태그와 textarea태그 둘 다 사용 가능합니다.
<input class="kigui-o-t" value="텍스트 박스"/>
<textarea class="kigui-o-t" value="텍스트 박스">텍스트 박스</textarea>

스위치

ON/OFF가 필요할 시 유용한 체크박스 입니다.
   
<label class="kigui-o-s" for="아이디">
    <input type="checkbox" id="아이디" />
    <div class="kigui-o-s-l"></div>
    <div class="kigui-o-s-b"></div>
</label>
ON/OFF 기본 상태는 input태그에 checked 속성을 추가하여야 합니다.
'아이디' 부분은 직접 적어주셔야 합니다.

체크 박스

일반적인 체크 박스 입니다.
   
<label class="kigui-o-c" for="아이디">
    <input type="checkbox" class="kigui-o-c" id="아이디"/>
    <div class="kigui-o-c-b">
        <i class="material-icons">check</i>
    </div>
    <div class="kigui-o-c-t">체크 박스</div>
</label>
ON/OFF 기본 상태는 input태그에 checked 속성을 추가하여야 합니다.
'아이디' 부분은 직접 적어주셔야 합니다.
